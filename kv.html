Ответы на контрольные вопросы по Git

Что такое «система контроля версий»?

Система контроля версий (Version Control System, VCS) — это инструмент, предназначенный для отслеживания изменений в коде и файлах проекта, позволяющий возвращаться к предыдущим версиям, отслеживать историю изменений и совместно работать над проектами.

К какому типу VCS относится Git? В чем главное отличие от других типов?

Git относится к распределённым системам контроля версий (DVCS). Главное отличие DVCS от централизованных систем в том, что каждая копия репозитория является полноценной и содержит всю историю изменений, что позволяет работать независимо от центрального сервера.

В чем заключается преимущество использования Git?

  • Надежность и высокая производительность.
  • Возможность работы в оффлайне, так как репозиторий является полностью локальным.
  • Гибкость и простота ветвления (branching) и слияния (merging).
  • Эффективная работа в команде и поддержка распределённых команд.

Какие команды Git вы знаете?

  • git init — инициализация нового репозитория.
  • git clone — клонирование существующего репозитория.
  • git add — добавление файлов в индекс.
  • git commit — создание коммита (фиксация изменений).
  • git push — отправка изменений на удалённый репозиторий.
  • git pull — получение изменений из удалённого репозитория.
  • git status — просмотр статуса текущего репозитория.

В чем отличие Git и GitHub?

Git — это система контроля версий, позволяющая управлять историей изменений проекта. GitHub — это веб-сервис (платформа), который предоставляет облачное хранилище для Git-репозиториев, позволяя разработчикам совместно работать над проектами через интернет.
