Answers to Git Control Questions

What is a Version Control System?

A Version Control System (VCS) is a tool designed to track changes in code and project files, allowing you to revert to previous versions, monitor the history of changes, and collaboratively work on projects.

What type of VCS is Git? What's its main difference from other types?

Git is a Distributed Version Control System (DVCS). The main difference between DVCS and centralized systems is that every repository copy is complete and contains the entire history of changes, allowing work independently from a central server.

What are the advantages of using Git?

  • Reliability and high performance.
  • Ability to work offline, as repositories are fully local.
  • Flexible and easy branching and merging.
  • Effective teamwork and support for distributed teams.

Which Git commands do you know?

  • git init – initialize a new repository.
  • git clone – clone an existing repository.
  • git add – add files to staging.
  • git commit – commit (record changes).
  • git push – push changes to a remote repository.
  • git pull – fetch changes from a remote repository.
  • git status – check the status of the current repository.

What is the difference between Git and GitHub?

Git is a version control system for managing project history and changes. GitHub is a web-based platform providing cloud storage for Git repositories, enabling developers to collaboratively work on projects online.
